# StS2 Concept Maps (컨셉 맵)

각 막의 지도에 **컨셉**(테마)을 부여하는 **슬레이 더 스파이어 2** 모드입니다 — *정예 시련장*, *이야기의 장*, *상인의 길* 처럼요. 방 종류와 경로(분기)가 그 컨셉에 맞춰 만들어져 맵 하나하나가 "성격"을 갖습니다. 컨셉엔 난이도 티어가 있어 **1막은 쉬운 컨셉, 3막으로 갈수록 어려운 컨셉**이 나옵니다.

> [Random Map], [Blind Map] 의 자매 모드. Random Map(모든 노드를 `?`로 가림)과 달리 Concept Maps는 **보이는 테마맵** — 구성을 미리 보고 경로를 짤 수 있습니다.

## 무엇을 하나

막 지도가 생성된 직후 컨셉을 하나 뽑아 노드 타입을 그 컨셉대로 **실제 배정**(보이는 테마맵)합니다. **경로**도 컨셉에 맞춰 다듬어집니다 — 선택형은 갈래가 많고, 강제형은 적으며, 컨셉마다 워크 방향이 약간씩 달라(좁게 vs 넓게) 위상도 컨셉별로 달라집니다.

STS2에는 독립된 "이벤트" 노드가 없고 `?`(Unknown) 타일이 곧 이벤트/미스터리 방이라, 컨셉의 "이벤트" 비중은 `?` 노드로 표현됩니다(진입 시 대부분 이벤트로 풀림).

### 컨셉 16종

| 티어 | 컨셉 | 성격 |
|---|---|---|
| 🟢 1 | 🧭 방랑자의 길 | 이벤트·휴식, 평온 |
| 🟢 1 | 🏪 시장 거리 | 상점(+이벤트) |
| 🟢 1 | 🌲 고요한 숲 | 일반몹+휴식, 정예 적음 |
| 🟢 1 | 🗺 보물의 길 | 보물 + 상점 약탈 루트 |
| 🟢 1 | 📖 이야기의 장 | 이벤트만 |
| 🟡 2 | ⚖ 균형 | 기본 분포 |
| 🟡 2 | 🔀 갈림길 | 일반몹+이벤트만 |
| 🟡 2 | 🏹 사냥터 | 일반몹 위주+휴식 |
| 🟡 2 | 💰 상인의 길 | 상점 천지 |
| 🟡 2 | 🎲 변덕의 땅 | 균일 카오스 |
| 🟡 2 | ⚔ 시험의 길 | 정예 섞인 전투 |
| 🔴 3 | 🛡 정예 시련장 | 정예만 |
| 🔴 3 | 🔥 시련의 불길 | 정예+몹, 휴식 적음 |
| 🔴 3 | ⚙ 분쇄기 | 전투 폭격 |
| 🔴 3 | 🏜 기근의 땅 | 휴식·상점 희박 |
| 🔴 3 | 💀 공포의 영역 | 3막 최난 |

막 게이팅: **1막** = 티어1 위주, **2막** = 티어2 중심, **3막+** = 티어3 위주.

### 공정성 안전망

배정은 노드별 순수 랜덤(지배 컨셉이 컨셉답게 뭉치도록)이고, 그 위에 몇 가지 최소 보장을 얹어 플레이 가능·테마 유지를 보장합니다.

- **휴식 가뭄 방지** — 맵마다 모닥불 최소 개수 보장(보스 직전 휴식은 별도로 항상 존재).
- **시그니처 보장** — 컨셉의 정체성 방 타입을 최소 비율 보장(운 나빠도 테마가 사라지지 않음).
- **보물 캡** — 보물 방은 진입 시 유물을 무료로 줍니다(바닐라는 막당 1개). 그래서 *추가* 보물 방 수에 상한을 두고, 초과분은 상점으로 전환(골드로 유물 구매)합니다. 약탈 루트를 넉넉히 주되 유물 홍수는 막습니다.

### 화면 배너

현재 맵 컨셉 이름을 난이도 티어색(초록/주황/빨강)으로 표시합니다. **드래그로 이동**(좌클릭, 위치 저장)하고 맵 노드 화면에서만 보입니다. 이름은 **16개 언어**로 현지화됩니다.

### 난이도 레벨 (Ⅰ / Ⅱ / Ⅲ)

맵마다 난이도 **레벨**도 뽑혀, 컨셉 이름 뒤에 로마 숫자로 표시됩니다(예: *Elite Gauntlet Ⅲ*). 레벨은 그 컨셉의 **패널티 강도**를 조절합니다 — **Ⅲ**는 패널티 최대(정예 시련장 Ⅲ는 일반몹 *0*), **Ⅰ**은 완화(조금 새어나옴). 레벨은 막 가중 랜덤이라 진행할수록 높아지되 변주가 있습니다(3막에 Ⅰ짜리 정예 시련장, 1막에 Ⅲ짜리 이야기의 장도 가능).

컨셉의 **티어**(언제 나오나=가용성)와는 별개입니다 — 티어=가용성, 레벨=강도.

### 보물지도(Spoils) 이벤트

보물지도 이벤트는 자체 모래시계 맵을 만드는데, 여기에도 컨셉이 적용됩니다 — 중앙 보물은 고정, 위/아래 경로만 테마화됩니다.

## 콘솔 명령

개발자 콘솔(모드 실행 중 활성)에서 `conceptmap` 명령으로 컨셉을 즉석 테스트:

| 명령 | 동작 |
|---|---|
| `conceptmap list` | 컨셉 목록 |
| `conceptmap <키>` | 해당 컨셉 강제 + 현재 막 재생성 |
| `conceptmap next` | 다음 컨셉으로 순환(전부 테스트) |
| `conceptmap surprise` | 자동(맵마다 컨셉)으로 복귀 |
| `conceptmap stats` | 마지막 맵의 패치/타입 내역 |

기본 모드는 **Surprise**(맵마다 자동 컨셉).

## 예외(원래 타입 유지)

- 시작 노드 / 보스(둘째 보스 포함) / 보스 직전 휴식(불) / 고정 보물(유물) 및 엔진 고정 노드.

## 결정성 / 멀티플레이

컨셉 선택과 노드/경로 배정 모두 런 시드+막으로 시드된 RNG라, 모든 플레이어(및 리로드)가 동일한 테마맵을 받습니다. 추가 네트워크 메시지 없음. 협동 시 전원 동일 버전 권장.

> **Concept Maps**와 **Random Map**을 동시에 쓰지 마세요 — 둘 다 맵 노드를 바꿉니다.

## 설치

1. [GitHub Releases](../../releases)에서 최신 `Sts2ConceptMap-vX.Y.Z.zip` 다운로드.
2. `Sts2ConceptMap/` 폴더를 `<슬더스2 설치경로>/mods/` 에 넣어 다음이 되게:
   ```
   <Slay the Spire 2>/mods/Sts2ConceptMap/Sts2ConceptMap.dll
   <Slay the Spire 2>/mods/Sts2ConceptMap/Sts2ConceptMap.json
   ```
3. 게임 실행.

## 소스 빌드

요구: .NET SDK 9.0, Godot.NET.Sdk 4.5.1(자동), 로컬 STS2 설치(자동 감지).

```sh
dotnet build Sts2ConceptMap.csproj -c Release
```

## 설정

- **끄기:** 실행 전 `STS2_CONCEPT_MAP_DISABLED=1`.
- **컨셉 고정(개발):** `STS2_CONCEPT_MAP_MODE=<키>` (예: `gauntlet`).
- **튜닝:** `Sts2ConceptMapCode/ConceptMapService.cs`의 카탈로그·보장·경로수·step편향·`MaxExtraTreasure` 수정 후 재빌드.
